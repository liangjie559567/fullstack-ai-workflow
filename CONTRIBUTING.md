# Contributing

感谢你改进这套 AI 全栈开发流程模板仓库。

## 提交前原则
1. 优先修改 `templates/` 与 `scripts/`，不要直接在 README 里描述一个并不存在的能力。
2. 任何模板变更都应同步更新：
   - `README.md`
   - `CHANGELOG.md`
   - `VERSION`（如有必要）
3. 任何脚本变更都应说明：
   - 输入
   - 输出
   - 失败条件
   - 是否会覆盖已有文件

## 提交流程
1. 新建分支
2. 修改模板 / 脚本 / 文档
3. 自查以下内容：
   - 模板路径是否一致
   - 脚本中的文件名是否与 templates/ 一致
   - README 示例命令是否仍然可用
4. 提交 PR

## PR 要求
请在 PR 描述中写清：
- 背景
- 修改内容
- 是否影响现有业务项目
- 是否需要升级 manifest / tag / 版本号
- 回滚方式

## 版本规则
- patch：文档修正、非破坏性小改动
- minor：新增模板、增强脚本、向后兼容扩展
- major：破坏性结构变更
